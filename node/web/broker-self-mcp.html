broker · self-mcp

The broker is its own MCP server.

Beyond tunnelling other servers, the broker exposes a reserved slot, _broker, that answers MCP itself — and an aggregate slot, _all, that unions every connected provider into one server.

Open the explorer demo → Docs
01 - The _broker slot

Introspect the broker over MCP

The reserved _broker slot is an in-process MCP server. Point any MCP client at it — MCP Inspector, Claude, your own — to inspect the broker's identity and the providers currently tunnelled in.

broker_info Reports the broker's identity, version and configuration.
providers_list Lists the provider slots currently registered with the broker.
provider_status Reports the connection status of a given provider slot.

Reachable on every client transport — see the endpoints below.

02 - The _all aggregate

One slot, every provider

The reserved _all slot presents the union of every opted-in provider's tools and prompts as a single MCP server. Aggregated names are provider-prefixed with a - separator, so a forecast tool from a weather provider appears as weather-forecast.

The broker's own introspection tools are aggregated in too: under _all they appear as _broker-broker_info, _broker-providers_list and _broker-provider_status. A client pinned to _all therefore sees the whole broker — its meta tools plus every tunnelled provider — through one connection.

Config-discovered upstreams (mcpServers and mcpbBundles) join _all automatically. Set "aggregate": false on an entry to keep it on its own slot only.
03 - Loading .mcpb bundles

Run a packaged server as a provider

The broker loads local .mcpb bundles listed in its config and runs them as stdio provider slots. Before a bundle is unpacked or spawned, the broker verifies a detached signature against a trusted public key — an unverified bundle is refused, never run.

JSON · .mcp-broker/config.json 
{
  "mcpbBundles": [{
    "name":       "weather",
    "path":       "bundles/weather.mcpb",
    "publicKey":  "bundles/mcpb-signing.pub.pem",
    "signature":  "bundles/weather.mcpb.sig",
    "aggregate":  true             // default; false keeps it off _all
  }]
}

Sign bundles with the bundled helper — Node built-ins only, no OpenSSL and no dependency on Anthropic's mcpb CLI:

Bash · from node/ 
# generate an Ed25519 signing key pair
node scripts/sign-bundle.mjs keygen bundles

# produce the detached signature next to the bundle
node scripts/sign-bundle.mjs sign bundles/weather.mcpb bundles/mcpb-signing.key.pem

Keep the private key out of version control. The broker only ever needs the public key referenced by publicKey.

04 - Endpoints

Where to connect

Both reserved slots are reachable on every client transport. Swap _broker for _all (or any provider slot name) in the paths below.

Broker · Streamable HTTP http://localhost:3000/_broker/mcp
Aggregate · Streamable HTTP http://localhost:3000/_all/mcp
Broker · Raw WebSocket ws://localhost:3000/_broker
Aggregate · Raw WebSocket ws://localhost:3000/_all
Broker · Legacy SSE http://localhost:3000/_broker/sse
Aggregate · Legacy SSE http://localhost:3000/_all/sse
